Validez XHTML

¿Qué es HXTML?

     XHTML (Lenguaje de Marcado de Hipertexto Extensible) es una versión más estricta y limpia de HTML , que nace precisamente con el objetivo de remplazar a HTML ante su limitación de uso con las cada vez más abundantes herramientas basadas en XML . XHTML extiende HTML 4.0 combinando la sintaxis de HTML, diseñado para mostrar datos, con la de XML, diseñado para describir los datos.

¿Qué documentos tienen validez XHTML?

      Tienen validez XHTML aquellos documentos HTML escritos como XML cuyos elementos cumplen estas características:

  • Han de estar correctamente anidados (nested).
  • Han de estar siempre cerrados (incluidos elementos vacios como las etiquetas de espaciado <hr/> <br/>...)
  • Han de estar escritos en minúsculas.
  • Y los elementos XHTML tienen que tener un elemento raíz (<html>).

¿Qué es Strict?

      Strict es un tipo de DTD (document type definition), que define un documento html o xml; strict como su nombre indica es algo estricto, permite elementos HTML y sus atributos, pero no incluye los elementos de presentación o desaprobamos o obsoletos (como la fuente). Los conjuntos de marcos no están permitidos.

¿Qué es transitional?

      Como Strict, Transitional es un tipo de DTD, más permisivo que el anterior este acepta tanto los elementos HTML y sus atributos como elementos de presentación y elementos obsoletos (como la fuente). En ambos los conjuntos de marcos no están permitidos.

¿Qué utiliza strict o transitional?

      Tienes que usar strict puesto que es un modelo admitido por la gran mayoria de navegadores y que ayuda a la cohesión del codigo de la pagina.
